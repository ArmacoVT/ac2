# American College Arcus Club — приложение, управление и качване

## Какво има

- `index.html` — приложението за **членове** (вход, 7-те формата, събития, резервации, профил).
- `admin.html` — **админ панелът** (събития, видимост, резервации).
- `db.js` / `config.js` — слой за данни и настройка на бекенда.
- `sw.js` / `manifest.json` / `icon-*.png` — правят го **инсталируемо приложение (PWA)**.
- `supabase/schema.sql` — базата данни.

Всички файлове стоят в **една папка**.

---

## 1) Демо режим (работи веднага, без нищо)

Докато `config.js` съдържа „PASTE…", приложението е в демо режим — данните се пазят в браузъра на устройството. Удобно за разглеждане. Вход: **admin / admin** (и в двата файла).

## 2) Свързване на Supabase (общо за всички устройства) — препоръчано

Това прави промените видими при всеки и въвеждането става веднъж.

1. Влез в **supabase.com → New project** (безплатно).
2. **SQL Editor → New query** → постави съдържанието на `supabase/schema.sql` → **Run**.
3. **Project Settings → API** → копирай **Project URL** и **anon public** ключа.
4. Отвори `config.js` и замени двете стойности:
   ```js
   SUPABASE_URL: "https://твоят-проект.supabase.co",
   SUPABASE_ANON_KEY: "eyJhbGci..."
   ```
5. Готово — приложението автоматично минава на „живо".

### Направи себе си администратор
1. Отвори приложението (`index.html`) → регистрирай се (имейл + парола + категория).
2. В Supabase → **SQL Editor** изпълни:
   ```sql
   update public.profiles set role='admin' where username='твоето_име';
   ```
3. Вече влизаш в `admin.html` със същия имейл/парола и имаш пълни права.

## 2б) Достъп само с покана (как се създават членове)

Регистрацията в приложението е **затворена** — никой не може сам да си направи акаунт. Ти каниш членовете:

1. Затвори публичната регистрация: Supabase → **Authentication → Providers → Email** → изключи **„Allow new users to sign up"**.
2. Покани член — **директно от админ панела** (бутон „Покани член" в раздел „Членове и видимост"). Въвеждаш имейл + категория и човекът получава покана да си зададе парола. Категорията се задава автоматично при поканата.
   - Алтернативно, без панела: Supabase → **Authentication → Users → Add user → Send invitation**.

### Еднократна настройка на бутона „Покани член"
Бутонът използва малка функция, която трябва да се качи веднъж:
1. Supabase → **Edge Functions → Create a function** → име **`invite-member`**.
2. Постави съдържанието на `supabase/functions/invite-member/index.ts` и натисни **Deploy**.
   (или с CLI: `supabase functions deploy invite-member`)
3. Готово — функцията автоматично има нужните ключове. Само администратор (profile.role='admin') може да кани.

### Забравена и смяна на парола
- На входа има **„Забравена парола?"** — членът въвежда имейла си и получава линк за нова парола.
- Влязъл в профила си, членът има бутон **„Смени парола"** по всяко време.
- (И двете работят само със свързан Supabase и включени имейли — Supabase ги праща наготово; за голям обем се настройва собствен SMTP.)

## 3) Управление от админ панела

- **Събития** — добавяш/редактираш/триеш; при всяко избираш **„Видимо за"** (всички или конкретни категории). Появява се само при членовете с право.
- **Резервации** — потвърждаваш / отказваш запитванията.
- **Видимост по членство** — обобщение кой какво вижда.

Категорията на всеки член се сменя в Supabase → **Table Editor → profiles → membership** (или role за админ).

### Live stream
Всяко събитие може да има **линк за live stream** и превключвател **„На живо сега"**. Кой може да гледа стрийма се определя от същото поле **„Видимо за"** — тоест всякакви комбинации (само Alumni, всички без Corporate, и т.н.).
- Поставяш линка (YouTube, Vimeo или друг) при събитието.
- Когато започне представлението, натискаш **„Пусни на живо"** (или отметката). Спираш със **„Спри стрийма"**.
- В приложението има раздел **„На живо"**, който е винаги видим: ако има активно излъчване и членът има достъп → отваря плеъра; ако няма → показва съобщение и следващото събитие; ако няма достъп за неговото членство → не се показва.

## 4) Качване онлайн + инсталиране на телефона (PWA)

1. Завлечи цялата папка `arcus-club-web` в **app.netlify.com** → получаваш `https://нещо.netlify.app`.
2. На телефона отвори адреса:
   - iPhone (Safari): Сподели → **Към началния екран**.
   - Android (Chrome): меню ⋮ → **Инсталиране на приложението**.
3. Появява се икона и се отваря на цял екран — като истинско приложение. Със свързан Supabase промените се отразяват при всички.

> Важно: PWA изисква адресът да е **https** (Netlify го дава автоматично). Тествай инсталирането от Netlify адреса, не от локален файл.

## 5) Native приложения (App Store + Google Play) — по-късно

Native проектът е в `../arcus-club-app` (Expo/React Native).
1. Сложи същите Supabase стойности в `arcus-club-app/app.json → expo.extra`.
2. Компилирай в облака:
   ```bash
   npm install -g eas-cli
   cd arcus-club-app && npm install
   eas build -p android   # .aab за Google Play
   eas build -p ios       # .ipa (нужен Apple Developer акаунт)
   ```
3. Публикуване: Google Play Console ($25 еднократно) и App Store Connect ($99/год).

## Сигурност

Със свързан Supabase входът е истински, паролите не са в кода, а видимостта на събитията се налага на ниво база (член не може дори да изтегли събитие, което не е за него). `admin / admin` важи само за демо режим.
